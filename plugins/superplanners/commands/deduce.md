# SuperPlanners - 分析推衍

对用户提出的问题进行分析推衍，给出最佳实践方案。

**核心规则：禁止修改任何文件，全程只读分析。**

分析目标：`{{{args}}}`

## 判断模式

检查参数中是否包含 `--deep`：
- 包含 `--deep` → 执行「深度思考模式」
- 不包含 → 执行「常规分析模式」

---

## 常规分析模式

### 第一步：理解问题

明确用户要分析的问题：
- 问题的核心是什么？
- 涉及哪些模块/文件/概念？
- 期望的分析方向（架构、性能、方案选型、重构...）

### 第二步：扫描相关代码

根据问题定位相关代码，只读取不修改：
- 找到相关文件和关键函数
- 理解现有实现逻辑和数据流
- 识别与问题相关的上下文（配置、依赖、调用关系）

### 第三步：分析推衍

基于代码现状进行分析：
- 当前实现的优势和不足
- 潜在的问题和风险点
- 可改进的方向

### 第四步：输出方案

给出最佳实践方案，格式如下：

```markdown
## 分析结论

{对问题的核心判断，1-3 句话}

## 方案对比

### 方案 A：{方案名}
- 思路：{简述}
- 优点：{列举}
- 缺点：{列举}
- 适用场景：{说明}

### 方案 B：{方案名}
- 思路：{简述}
- 优点：{列举}
- 缺点：{列举}
- 适用场景：{说明}

（如有更多方案继续列举）

## 推荐

推荐方案 {X}，理由：{具体原因}

## 实施要点

（如果用户后续要实施，需要注意的关键点）
```

## 深度思考模式（--deep）

当参数包含 `--deep` 时，在常规分析的基础上，额外调用 `mcp__sequential-thinking__sequentialthinking` 工具进行多步链式推理。

### 第一步：问题定义

调用 sequential-thinking，初始思考：
- 精确定义问题边界
- 列出需要回答的子问题
- 识别关键假设

### 第二步：扫描相关代码

同常规模式，只读取相关代码。

### 第三步：链式深度推理

持续调用 sequential-thinking 进行多轮推理：

1. **假设阶段**：提出可能的方案假设
2. **验证阶段**：结合代码事实验证每个假设，标记需要修正的地方
3. **修正阶段**：根据验证结果修正假设，深入分析
4. **多角度评估**：从以下维度逐一分析
   - 性能影响
   - 可维护性
   - 扩展性
   - 安全性
   - 实施成本
5. **收敛结论**：综合所有分析得出最终判断

### 第四步：输出完整推衍报告

```markdown
## 推衍过程

### 问题定义
{精确的问题描述和边界}

### 关键假设
1. {假设 1} → 验证结果：{成立/不成立/部分成立}
2. {假设 2} → 验证结果：{成立/不成立/部分成立}
...

### 推理链
{思考步骤 1} → {思考步骤 2} → ... → {结论}
（标注哪些步骤进行了修正及修正原因）

### 多维度评估

| 维度 | 方案 A | 方案 B | ... |
|------|--------|--------|-----|
| 性能 | {评分和说明} | {评分和说明} | |
| 可维护性 | {评分和说明} | {评分和说明} | |
| 扩展性 | {评分和说明} | {评分和说明} | |
| 安全性 | {评分和说明} | {评分和说明} | |
| 实施成本 | {评分和说明} | {评分和说明} | |

## 最终结论

{综合推衍后的最佳实践方案}

## 推荐

推荐方案 {X}，理由：{基于推衍过程的具体原因}

## 实施要点

{关键注意事项}
```
